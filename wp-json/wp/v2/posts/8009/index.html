Hi everyone!<\/p>\n

We are back with another edition of “What’s new in Elixir”. This time we are working towards Elixir v1.8 which should be out next January. Here are some highlights of what have already made to master:<\/p>\n

    \n
  • mix test<\/code> now supports the --max-failures<\/code> flag (commit<\/a>)<\/p>\n<\/li>\n
  • \n

    The Inspect<\/code> protocol can now be easily derivable and this is a great mechanism to exclude some fields during inspection and logs, especially security-related ones (commit<\/a>)<\/p>\n<\/li>\n

  • \n

    Mix now automatically starts optional dependencies when using the :extra_applications<\/code> configuration (commit<\/a>)<\/p>\n<\/li>\n

  • \n

    We finally have official support for time zone databases in Elixir. Elixir still handles only UTC by default but you can bring another database to use the DateTime<\/code> API with other time zones (commit<\/a>)<\/p>\n<\/li>\n

  • \n

    There are many other small additions, such as List.myers_difference\/3<\/code><\/a> and String.bag_distance\/2<\/code><\/a> which were extracted from the Elixir codebase itself<\/p>\n<\/li>\n<\/ul>\n

    For the complete list of changes, see the CHANGELOG<\/a>. To keep track of upcoming changes and even contribute some features yourself, head to Elixir\u2019s issues tracker<\/a>.<\/p>\n

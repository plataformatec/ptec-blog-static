There is a vulnerability in Devise source code that allows someone to steal your session through session fixation attacks.<\/p>\n

Who is affected?<\/h3>\n

This vulnerability is present in all Devise versions, in both 1.0 and 1.1 branches. However, you are only affected if you are using a Active Record ou Memcached or other server persistent session store. Projects using cookie stores (Rails default) are not affected.<\/p>\n

If you are using Devise defaults which requires the user a password to update his account, it is unlikely the target account can be stolen, however the attacker will be able to normally perform actions in the website.<\/p>\n

Releases<\/h3>\n

To fix this vulnerability, we released Devise 1.0.9 for Rails 2.3.x applications and Devise 1.1.4 for Rails 3.x.<\/p>\n

In another note, Devise 1.2.rc was also released, which includes this security fix and Omniauth support. Check the CHANGELOG<\/a> for more information.<\/p>\n

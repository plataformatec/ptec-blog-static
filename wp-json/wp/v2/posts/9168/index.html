In 2017 Hex.pm got support for Private packages and organizations<\/a>, a way for teams to publish and manage packages without making them public. While this works great for many organizations, some have stricter compliance requirements and need to host packages on their own infrastructure.<\/p>\n

Today we are happy to announce MiniRepo<\/a>, a minimal Hex server that can be used for packages self-hosting.<\/p>\n

MiniRepo ships with the following features:<\/p>\n

    \n
  • Pluggable storage (with built-in adapters for local filesystem and Amazon S3)<\/li>\n
  • Mirroring<\/li>\n
  • Publishing packages via HTTP API<\/li>\n
  • Support for multiple repositories and mirrors<\/li>\n<\/ul>\n

    The goal of MiniRepo is to be minimal so that it can be easy to understand and can serve as a starting point for a more complete solution that may make sense in a given organization.<\/p>\n

    See instructions for usage with Mix<\/a> and Rebar3<\/a>.<\/p>\n

    Finally, by making it easier to run self-hosted Hex registry we are achieving one of the goals of the Building and Packaging Working Group<\/a> at Erlang Ecosystem Foundation<\/a>, which we are glad to contribute to!<\/p>\n","protected":false},"excerpt":{"rendered":"
